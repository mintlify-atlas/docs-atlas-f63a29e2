<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-3xl font-bold text-gray-900 dark:text-white mb-4">
    Quick start
  </h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
    Get your proxy server running on Heroku in minutes
  </p>
  
  <Steps>
    <Step title="Deploy to Heroku">
      Click the Deploy to Heroku button and configure your environment variables. The one-click deployment sets up everything automatically.
      
      ```bash
      # Or clone and deploy manually
      git clone https://github.com/CoolFool/roxy
      cd roxy
      heroku create
      ```
    </Step>
    
    <Step title="Configure your domain">
      Set the `DOMAIN` environment variable to the website you want to proxy through Heroku.
      
      ```bash
      heroku config:set DOMAIN=example.com
      ```
      
      <Note>
        If a website redirects to a different domain (e.g., `google.com` → `www.google.com`), use the redirected domain as your `DOMAIN` value.
      </Note>
    </Step>
    
    <Step title="Optional: Enable keep-alive">
      Prevent your Heroku free dyno from sleeping after 20 minutes of inactivity.
      
      ```bash
      heroku config:set KEEP_DYNO_ALIVE=true
      heroku config:set HEROKU_APP_URL=https://your-app-name.herokuapp.com
      ```
    </Step>
    
    <Step title="Access your proxy">
      Visit your Heroku app URL to start using the proxy. All requests will be forwarded to your configured domain with proper URL rewriting.
      
      ```bash
      heroku open
      ```
    </Step>
  </Steps>
</div>
